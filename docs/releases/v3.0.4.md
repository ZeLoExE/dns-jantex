## DNS Jantex 3.0.4 — Stability & Security Update

### Key Changes

- UI runs without Administrator privileges
- UAC is requested only for Apply, Reset, and Flush via a scoped elevated helper
- Smart Connect now uses real DNS queries instead of ICMP ping
- Samples three measurements and selects the median for more stable results
- Fixed Auto Flush; it now runs only when enabled and exactly once
- Language, selected provider, and update status are preserved after restart
- Settings are stored in `%LOCALAPPDATA%\DNS Jantex`
- Atomic JSON writes with backup of corrupt files for recovery
- Adapter selection based on the Windows default route
- Fixed duplicate providers and Mullvad translation key
- Worker threads shut down safely without `QThread.terminate()`

### Auto Update

This release includes two required assets:

- `DNSJantex-Setup.exe`
- `DNSJantex-checksums.json`

Newer versions verify the installer file size and SHA-256 hash before execution.
If a Code Signing Certificate is added later, Authenticode verification will also be enforced simultaneously.

### Installation Note

Users running older versions can use **Check for Updates** to upgrade.
Windows will only show the UAC prompt when running the installer or changing DNS settings.
